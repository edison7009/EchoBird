# 26. 通用 CLI Bridge 技术路线

> **状态：✅ 核心功能已完成 · 分支：`feature/cli-bridge`**  
> 目标：让 Echobird 的 Bridge 能连接任意 AI CLI 工具，并为 Agent 提供角色身份选择。

---

## 一、功能概述

### 1. 多 CLI 通信

通过统一的通信层连接多种 Agent CLI 工具（本地 + SSH 远程）：

| CLI | 厂商 | 协议 | 状态 |
|---|---|---|---|
| OpenClaw | EchoBird | JSON stream over PTY | ✅ 已支持 |
| Claude Code | Anthropic | `claude --print` one-shot | ✅ 已支持 |
| OpenCode | OpenCode.ai | headless one-shot | ✅ 已支持 |
| ZeroClaw | ZeroClaw Labs | headless one-shot | ✅ 已支持 |
| 通用 CLI | — | 原始文本 I/O | 兜底方案 |

### 2. Agent 角色身份系统（✅ 已完成）

基于 [agency-agents](https://github.com/msitarzewski/agency-agents)（MIT 协议）提供的 150+ 专业角色：

**角色选择器 UI：**

- 顶部 Agent 工具 Tab（OpenClaw / Claude Code / OpenCode / ZeroClaw）
  - 未安装的 Agent **灰色不可点击**（通过 `detect_local_agents` 检测）
- 分类 Tab 筛选（全部 / 工程部 / 设计部 / 营销部 / 产品部 等 12+ 分类）
- 第一个卡片为🚫**"取消角色"**卡（`none.jpg`），点击清除当前角色
- 6 列竖版卡片网格（5:8 比例），图片从 CDN 加载 + 骨架屏
- 悬停滑入动画（名字固定，描述展开）
- 点击即选（✓ 角标），无需二次确认
- 选中后频道输入框上方显示角色名，取消后显示工具名

**数据存储：** 2 个 JSON 文件（`roles-en.json` + `roles-zh-Hans.json`），不再扫描 .md 目录。

### 3. 角色交易平台与加密保护（📋 规划中）

> 选择角色 > 安装技能。角色 = 雇佣一个熟练工，优质角色 Prompt 是核心知识产权。

**商业模式：**

| 类型 | 来源 | 加密 | 价格 |
|---|---|---|---|
| 默认角色集 | 随 Echobird 内置 | ❌ 明文 | 免费 |
| 付费角色 | 官方 / 第三方创作者 | ✅ 加密 | 按角色定价 |

**架构预留：**

1. JSON 文件预留 `license_type` 字段扩展空间
2. Bridge CLI 预留 `decrypt_role()` 函数接口
3. 角色选择器 UI 预留付费标识（如 💎 角标）

## 二、架构设计

```
前端 Channels 页面
    ↓  tauri invoke: bridge_start / bridge_chat / bridge_stop
Bridge Manager (Rust, channel_commands.rs)
    ↓  根据 plugin.json 的 protocol 字段动态分派
    ├── stdio-json     → Bridge Binary 模式 (OpenClaw)
    ├── cli-oneshot    → 每次聊天 spawn CLI 子进程 (Claude Code, OpenCode)
    └── cli-interactive → 持久 PTY 子进程 (通用 CLI, Phase 3)
```

### 协议类型

| protocol | 描述 | 生命周期 |
|---|---|---|
| `stdio-json` | 通过 bridge binary (stdin/stdout JSON) 持久通信 | 持久进程 |
| `cli-oneshot` | 每次聊天 spawn CLI 子进程执行一次，读 stdout | 一次性 |
| `cli-interactive` | 持久 PTY 子进程 + ready_signal 检测 | 持久进程 |

## 三、Plugin 配置格式

每个 CLI 工具用 `plugin.json` 描述，存放于 `plugins/{id}/`：

```json
{
    "id": "claudecode",
    "name": "Claude Code",
    "protocol": "cli-oneshot",
    "cli": {
        "command": "claude",
        "detectCommand": "claude --version",
        "args": ["--print", "--output-format", "text"],
        "resumeArgs": ["--print", "--continue", "{sessionId}"],
        "sessionArg": "--session-id",
        "sessionMode": "always",
        "systemPromptArg": "--system-prompt",
        "systemPromptWhen": "new-session"
    }
}
```

## 四、角色数据架构（✅ 已实现）

### 设计原则

- **2 个 JSON 文件**：不扫描 .md 目录，直接读 JSON 元数据
- **CDN 图片**：封面图从 `echobird.ai/docs/roles/` 加载，不打包进客户端
- **骨架屏 + Fallback**：加载时显示 pulse 动画，CDN 失败回退本地占位图

### 角色数据来源

| 来源 | 角色数 | 语言 | 许可证 |
|---|---|---|---|
| [agency-agents](https://github.com/msitarzewski/agency-agents) | 156 | 英文 | MIT |
| [agency-agents-zh](https://github.com/jnMetaCode/agency-agents-zh) | 151（含中国原创） | 中文 | MIT |

### 文件结构

```
roles/
├── roles-en.json        ← 156 roles, 13 categories（前端直读）
├── roles-zh-Hans.json   ← 151 roles, 12 categories（前端直读）
├── en/                  ← .md 原始文件（仅 Bridge CLI 远程部署用）
└── zh-Hans/             ← .md 原始文件（仅 Bridge CLI 远程部署用）
```

### JSON 格式

```json
{
  "allLabel": "全部",
  "categories": [
    { "id": "engineering", "label": "工程部", "order": 0 }
  ],
  "roles": [
    {
      "id": "engineering-frontend-developer",
      "name": "前端开发者",
      "description": "精通 React/Vue...",
      "category": "engineering",
      "filePath": "engineering/engineering-frontend-developer.md",
      "img": "https://echobird.ai/docs/roles/7.jpg"
    }
  ]
}
```

### CDN 图片架构

```
echobird.ai/docs/roles/
├── none.jpg          ← "取消角色"卡封面
├── 1.jpg ~ 17.jpg    ← 演示用占位图（后续替换为正式角色封面）
```

前端加载流程：
1. 打开选择器 → 骨架屏（`animate-pulse` 渐变）
2. `<img>` lazy load CDN 图片 → 加载完 fade-in 0.3s
3. CDN 失败 → `onError` 自动 fallback 到本地 `/role/{n}.jpg`

## 五、Agent 检测与角色管理（✅ 已实现）

### 本地 Agent 检测

Rust `detect_local_agents()` 命令，跨平台检测 4 个 Agent CLI：

| 平台 | 检测命令 |
|---|---|
| **Windows** | `where.exe claude / opencode / openclaw / zeroclaw` |
| **Linux / macOS** | `which claude / opencode / openclaw / zeroclaw` |

返回数据：
```json
[
  { "id": "claudecode", "name": "Claude Code", "installed": true, "path": "C:\\...\\claude.exe" },
  { "id": "zeroclaw",   "name": "ZeroClaw",   "installed": false, "path": null }
]
```

角色选择器打开时并行调用，未安装的 Agent Tab 灰色 + `cursor-not-allowed` + tooltip "未安装"。

### Bridge CLI 角色命令（✅ 已实现）

| 消息类型 | 说明 | 状态 |
|---|---|---|
| `detect_agents` | 检测已安装的 Agent CLI | ✅ |
| `set_role` | 检查角色文件 → 不存在则拷贝到 Agent 目录 | ✅ |
| `clear_role` | 删除 Agent 目录下的角色文件/目录 | ✅ |
| `start_agent` | 启动指定 Agent CLI 进程 | ✅ |
| `chat` | 发送消息给 Agent | ✅ |

### 各 Agent 角色安装/清除路径

| Agent | 路径 | 类型 |
|---|---|---|
| Claude Code | `~/.claude/agents/{role}.md` | 文件 |
| OpenCode | `~/.config/opencode/agents/{role}.md` | 文件 |
| OpenClaw | `~/.openclaw/agency-agents/{role}/` | 目录 |
| ZeroClaw | `~/.zeroclaw/workspace/skills/{role}/` | 目录 |

## 六、角色注入流程

**Echobird 只传 role_id，Bridge CLI 自己处理文件分发。**

```
Echobird UI (本地)                    Bridge CLI (远程服务器)
   │                                      │
   │  选择角色                              │
   │  ── set_role(agent_id, role_id) ──►   │
   │                                      ├── 读取 roles/ 下对应 .md
   │                                      ├── 拷贝到 Agent 原生目录
   │  ◄──── role_set { installed: true } ─  │
   │                                      │
   │  取消角色                              │
   │  ── clear_role(agent_id, role_id) ─►  │
   │                                      ├── 删除 Agent 目录下角色文件
   │  ◄──── role_cleared { success: true } │
```

### 消息发送流程

```
用户点击「发送」
     │
     ├── ① 检测 Agent 进程 → 没运行则自动启动
     ├── ② 检查角色文件 → 不存在则拷贝（幂等）
     └── ③ 发送消息 → 返回响应
```

关键原则：自动激活、不重复启动、幂等角色安装。
